# Selector Strategy

## Priority Order

When choosing a CSS selector for `[UnionInit]`, use the **highest-priority selector available** in the current markup:

| Priority | Selector Type | Example | Stability |
|----------|--------------|---------|-----------|
| 1 | aria attributes | `[aria-label='Submit']`, `[role='dialog']` | Highest — tied to accessibility contract |
| 2 | data-testid | `[data-testid='submit-btn']` | High — dedicated testing attribute |
| 3 | ID | `#submit-button` | High — but may conflict with JS/CSS coupling |
| 4 | CSS class | `.submit-btn`, `.theme-btn.primary` | Medium — may change with styling |
| 5 | Text-based | `text=Submit`, `:has-text('Submit')` | Medium — changes with i18n/copy |
| 6 | :has() pseudo | `.form:has(.error)` | Lower — depends on DOM structure |
| 7 | Attribute | `[type='submit']`, `[data-id]` | Varies — depends on attribute purpose |

## When to Use Each

**aria attributes** (priority 1):
```csharp
// Best: uses accessibility contract — survives refactoring
[UnionInit("[aria-label='Close dialog']")]
public UnionElement CloseButton { get; set; }

[UnionInit("[role='dialog']")]
public UnionElement DialogRoot { get; set; }
```
Use when the app markup has aria attributes. These are the most stable because they're tied to the accessibility contract — changing them would break screen readers, so they rarely change unintentionally.

**data-testid** (priority 2):
```csharp
[UnionInit("[data-testid='company-search']")]
public UnionElement SearchInput { get; set; }
```
Use when no aria attribute exists but the team has added test-specific attributes. These exist solely for testing and are stable by convention.

**ID** (priority 3):
```csharp
[UnionInit("#username")]
public UnionElement EmailInput { get; set; }

[UnionInit("#log_in")]
public UnionElement LoginButton { get; set; }
```
Good when IDs exist and are stable. Watch for IDs generated by frameworks (e.g., Blazor's `_blazor-id-*`) — these are not stable.

**CSS class** (priority 4):
```csharp
[UnionInit(".toaster__icon--success")]
public UnionElement SuccessToast { get; set; }

[UnionInit(".select-company-wrapper")]
public UnionElement CompanySelection { get; set; }
```
Prefer BEM-style classes (`.block__element--modifier`) over utility classes (`.mt-4`, `.flex`). Utility classes change frequently.

**Text-based** (priority 5):
```csharp
[UnionInit(".theme-btn:has-text('Continue')")]
public UnionElement ContinueButton { get; set; }

[UnionInit("text=No companies")]
public UnionElement NoCompaniesMessage { get; set; }
```
Use when no structural selector is available. Fragile with i18n — text changes break these.

**:has() and complex selectors** (priority 6):
```csharp
[UnionInit(".select-company-option-wrap:has(p:text-is(\"Acme Corp\"))")]
public UnionElement AcmeCompanyRow { get; set; }
```
Use for ItemBase patterns where identifying items by content is necessary. Avoid for static elements.

## Selector Anti-Patterns

```csharp
// WRONG: Deep nesting — fragile, breaks with DOM restructuring
[UnionInit("div > div > ul > li:nth-child(2) > span.name")]
public UnionElement SecondItem { get; set; }

// WRONG: Utility classes — change with styling
[UnionInit(".mt-4.flex.justify-between")]
public UnionElement Header { get; set; }

// WRONG: Framework-generated IDs — not stable
[UnionInit("#_blazor-id-42")]
public UnionElement SomeElement { get; set; }

// WRONG: Overly broad — may match multiple elements
[UnionInit("button")]
public UnionElement Button { get; set; }
```

## Aria Migration

When the application adds aria attributes to elements that currently use lower-priority selectors, update the `[UnionInit]` selectors:

```csharp
// Before: CSS class selector
[UnionInit(".submit-btn")]
public UnionElement SubmitButton { get; set; }

// After: aria attribute available
[UnionInit("[aria-label='Submit form']")]
public UnionElement SubmitButton { get; set; }
```

Look for migration opportunities when:
- The frontend team adds accessibility attributes to components
- A PR includes `aria-*` or `role=` attributes on elements you have selectors for
- You're creating new page objects for pages that already have aria markup

## Combining Selectors

When a single selector isn't specific enough, combine them:

```csharp
// Combine class + text for specificity
[UnionInit(".theme-btn:has-text('Continue')")]
public UnionElement ContinueButton { get; set; }

// Combine class + attribute
[UnionInit(".company-list [data-status='active']")]
public UnionElement ActiveCompanies { get; set; }

// OR logic for elements that may render differently
[UnionInit(".link_here, a:has-text('Create company here')")]
public UnionElement CreateCompanyLink { get; set; }
```
